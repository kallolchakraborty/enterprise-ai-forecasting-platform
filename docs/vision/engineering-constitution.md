# Engineering Constitution

These principles are non-negotiable.

## 1. Learning First

Every implementation must teach a concept.

---

## 2. Documentation First

No feature is complete until it is documented.

---

## 3. API First

Every capability is exposed through a well-designed FastAPI endpoint.

---

## 4. Clean Architecture

Business logic never depends on frameworks.

---

## 5. Test Everything

Every feature includes automated tests.

---

## 6. Explain Everything

Every important architectural decision includes an explanation and trade-off analysis.

---

## 7. Production Mindset

Even learning examples should follow production-quality practices where practical.

---

## 8. Build for Change

Design modules to evolve without requiring major rewrites.

---

## 9. Open Source Quality

Every commit should improve the repository in a way that others can learn from.

---

## 10. Continuous Improvement

Architecture is expected to evolve as new insights are gained.
